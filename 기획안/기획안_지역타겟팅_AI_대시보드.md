# 지역 기반 영업 타겟팅 AI 대시보드 기획안

## 1. 서비스 개요

**한 줄 정의**: 법정동별 소비력·인구·부동산 데이터를 결합하여 영업 잠재력을 스코어링하고, 대화형 AI 에이전트로 인사이트를 즉시 질의할 수 있는 대시보드

**사용자**: 아정당 영업기획팀, 마케팅팀, 지역 영업 담당자

**핵심 가치**: "감으로 하던 지역 타겟팅을 데이터 기반 의사결정으로 전환"

## 2. 대시보드 구성 (4개 섹션)

### 섹션 A: 지역 잠재력 스코어 지도

#### 화면 구성
- 서울 법정동 단위 코로플레스(Choropleth) 지도
- 법정동 클릭 시 상세 패널 오픈
- 상단 필터: 기간(월), 카테고리(인터넷/렌탈/모바일), 스코어 유형

#### 핵심 지표: **영업 잠재력 스코어 (0~100)**

계산 방식:

```
잠재력 스코어 = (수요 스코어 × 0.4) + (구매력 스코어 × 0.3) + (미개척도 스코어 × 0.3)
```

**① 수요 스코어 (Demand Score, 0~100)**
- 데이터 소스: SPH 유동인구 + 리치고 부동산 시세
- 계산:
  ```
  이사수요지수 = 해당 법정동의 최근 3개월 전세 평단가 변동률 (절대값)
  인구유입지수 = 해당 법정동의 방문인구 전월 대비 증감률
  신규가구지수 = 20~30대 인구 비율 × 1인가구 추정치
  
  수요 스코어 = normalize(이사수요지수 × 0.4 + 인구유입지수 × 0.3 + 신규가구지수 × 0.3)
  ```
- 정규화: 전체 법정동 중 상대적 위치 (Min-Max → 0~100)

**② 구매력 스코어 (Purchasing Power Score, 0~100)**
- 데이터 소스: SPH 자산소득
- 계산:
  ```
  소득지수 = 해당 법정동의 평균소득 / 서울 전체 평균소득
  카드이용지수 = 1인당 카드이용금액 (법인 제외, 개인만)
  자산지수 = 평균 자산 수준 (부동산 + 금융)
  
  구매력 스코어 = normalize(소득지수 × 0.4 + 카드이용지수 × 0.3 + 자산지수 × 0.3)
  ```

**③ 미개척도 스코어 (Untapped Potential Score, 0~100)**
- 데이터 소스: 아정당 V01 지역별 계약 + SPH 세대수
- 계산:
  ```
  현재침투율 = 해당 시군구의 최근 6개월 계약건수 / 추정 세대수
  미개척도 = 1 - 현재침투율 (높을수록 기회가 많음)
  
  미개척도 스코어 = normalize(미개척도)
  ```
- 참고: 아정당 데이터가 시군구 단위이므로, 같은 구 내 법정동은 동일 침투율 적용 후 세대수로 가중

#### 법정동 상세 패널 (클릭 시)
| 항목 | 데이터 소스 | 표시 방식 |
|------|-----------|----------|
| 잠재력 스코어 | 계산값 | 게이지 차트 (0~100) |
| 인구 구조 | 리치고 성별연령별 | 인구 피라미드 차트 |
| 소득 수준 | SPH 자산소득 | 소득구간 분포 바 차트 |
| 카드 소비 패턴 | SPH 카드매출 | 업종별 매출 비중 도넛 차트 |
| 부동산 시세 추이 | 리치고 시세 | 매매/전세 라인 차트 (13년치) |
| 유동인구 패턴 | SPH 유동인구 | 시간대별 거주/직장/방문 스택바 |
| 과거 계약 실적 | 아정당 V01 | 월별 계약건수 라인 차트 |

### 섹션 B: 상품 추천 매트릭스

#### 화면 구성
- 시군구 × 상품 카테고리 히트맵
- 색상: 추천 강도 (진할수록 해당 지역에 해당 상품 적합)

#### 추천 로직

**인터넷 결합상품 추천**
```
IF 세대수 > 중앙값 AND 평균소득 상위 30%:
    → 프리미엄 결합 (인터넷+TV+WiFi+전화) 추천
    근거: 구매력 높은 대가구 지역

IF 20~30대 비율 > 40% AND 1인추정가구 높음:
    → 단독가입 또는 유심만 추천
    근거: 가성비 선호 1인가구 지역

IF 40~50대 비율 > 30% AND 평균소득 중위 이상:
    → 투게더결합 / 요즘가족결합 추천
    근거: 가족 단위 결합상품 적합
```

**렌탈 상품 추천**
```
IF 5세미만 대비 2040여성 비율 > 0.15 (영유아 가구 밀집):
    → 정수기 + 공기청정기 패키지 추천
    근거: 영유아 가구 필수 렌탈

IF 60대 이상 비율 > 20%:
    → 안마의자 + 비데 추천
    근거: 시니어 라이프스타일 매칭

IF 카드매출 중 음식+커피 비중 상위 AND 20~30대 밀집:
    → 스타일러 + 건조기 추천
    근거: 외식 중심 젊은층, 생활가전 렌탈 선호
```

#### 데이터 기반 검증 방법
- 아정당 V06(렌탈 카테고리 트렌드)의 시도별 실적과 비교
- V02(서비스 결합 패턴)의 실제 결합 비율로 추천 정확도 백테스트

### 섹션 C: 마케팅 채널 퍼포먼스

#### 화면 구성
- 좌측: 채널별 퍼널 Sankey 다이어그램 (유입 → 상담 → 접수 → 개통 → 결제)
- 우측: 채널별 KPI 테이블

#### 핵심 지표 및 계산

| 지표 | 계산식 | 데이터 소스 |
|------|--------|-----------|
| 채널별 CPA (건당 획득비용) | 총 마케팅비 / TOTAL_CONTRACTS | V07 (추정, 매출 데이터 역산) |
| 채널별 ROAS | TOTAL_REVENUE / 추정 광고비 | V07 |
| 퍼널 단계별 이탈률 | 1 - (다음단계 / 현재단계) | V03 |
| 채널 × 카테고리 전환율 | PAYEND_COUNT / CONTRACT_COUNT | V04 |
| 디바이스별 전환율 | CONVERSION_EVENT_COUNT / SESSION_COUNT | V08 |
| 시간대별 콜 효율 | CONNECTED_COUNT / CALL_COUNT by HOUR | V10 |

#### 인사이트 자동 생성 (규칙 기반)
```
규칙 1: IF 채널의 세션수 상위 20% BUT 전환율 하위 20%
    → "⚠️ {채널명}: 트래픽은 많으나 전환 부진. 랜딩페이지 또는 타겟팅 점검 필요"

규칙 2: IF 전월 대비 전환율 20% 이상 하락
    → "📉 {채널명}: 전환율 급락 감지 ({이전}% → {현재}%). 광고 소재 피로도 확인"

규칙 3: IF 수신콜 전환율 > 발신콜 전환율 × 1.5
    → "📞 수신콜 전환율이 발신 대비 {n}배 높음. 인바운드 유도 캠페인 강화 권장"
```

#### 에이전트가 답할 수 있는 질문 유형 및 처리 방식

**유형 1: 지역 진단**
```
Q: "강남구 역삼동은 어떤 지역이야?"
처리:
  1. region_master에서 역삼동의 district_code 조회
  2. 병렬 쿼리:
     - SPH 자산소득 → 평균소득, 직업군 분포
     - SPH 유동인구 → 거주/직장/방문 비율
     - SPH 카드매출 → 상위 소비 업종 3개
     - 리치고 시세 → 최근 매매/전세 평단가
     - 리치고 인구 → 연령대 분포
  3. 종합 프로파일 생성:
     "역삼동은 직장인구 비중이 76%로 오피스 밀집 지역입니다.
      평균소득 상위 15%, 카드매출은 음식>커피>의류 순입니다.
      20~30대 비율 38%, 매매 평단가 5,200만원/평입니다.
      → 인터넷 단독가입 + 커피머신 렌탈 타겟팅 추천"
```

**유형 2: 비교 분석**
```
Q: "중구랑 영등포구 중에 렌탈 영업하기 좋은 곳은?"
처리:
  1. 두 구의 district_code 목록 조회
  2. 비교 쿼리:
     - 인구 구조 비교 (영유아 비율, 가구 추정)
     - 소득/카드 소비력 비교
     - 아정당 V01 과거 렌탈 계약 실적 비교

  3. 비교 테이블 + 결론:
     "영등포구가 렌탈에 더 적합합니다.
      - 세대수 2.3배 (12만 vs 5.2만)
      - 5세미만/2040여성 비율: 영등포 0.15 vs 중구 0.09
      - 과거 계약건수도 영등포가 1.8배
      단, 중구는 평균 순매출이 12% 높아 고가 렌탈 가능성 있음"
```

**유형 3: 트렌드/이상치**

## 3. 데이터 파이프라인

### 테이블 설계 (DuckDB/SQLite)

```sql
-- 지역 매핑 마스터 (SPH + 리치고 + 아정당 연결)
CREATE TABLE region_master (
    district_code    VARCHAR(8),    -- SPH 기준
    bjd_code         VARCHAR(10),   -- 리치고 기준
    province_name    VARCHAR(20),   -- "서울특별시"
    province_short   VARCHAR(10),   -- "서울" (아정당/리치고 조인용)
    city_name        VARCHAR(20),   -- "중구"
    district_name    VARCHAR(20),   -- "신당동"
    geometry         TEXT           -- GeoJSON (지도 렌더링용)
);

-- SPH 유동인구 (집계 뷰)
CREATE VIEW v_floating_population AS
SELECT district_code, standard_year_month,
       SUM(residential_population) as total_resident,
       SUM(working_population) as total_working,
       SUM(visiting_population) as total_visiting
FROM sph_floating_population
GROUP BY district_code, standard_year_month;

-- SPH 카드매출 (집계 뷰)
CREATE VIEW v_card_sales AS
SELECT district_code, standard_year_month,
       SUM(CASE WHEN card_type='1' THEN total_sales_amount END) as personal_sales,
       SUM(CASE WHEN card_type='1' THEN food_amount END) as food_sales,
       SUM(CASE WHEN card_type='1' THEN coffee_amount END) as coffee_sales
       -- ... 20개 업종
FROM sph_card_sales
WHERE card_type = '1'  -- 개인만
GROUP BY district_code, standard_year_month;

-- 잠재력 스코어 (계산 뷰)
CREATE VIEW v_potential_score AS
SELECT
    r.district_code,
    r.city_name,
    r.district_name,
    -- 수요 스코어 요소
    jeonse_change_rate,
    population_inflow_rate,
    young_ratio,
    -- 구매력 스코어 요소
    avg_income_ratio,
    card_usage_per_capita,
    -- 미개척도 스코어 요소
    1 - penetration_rate as untapped,
    -- 종합 스코어
    (demand_norm * 0.4 + purchase_norm * 0.3 + untapped_norm * 0.3) * 100 as potential_score
FROM region_master r
JOIN ... ;
```

### CSV → DB 적재 순서
1. SPH 법정동 마스터 → `region_master` (GeoJSON 파싱 포함)
2. 리치고 3개 파일 → `richgo_*` 테이블 + `region_master` BJD_CODE 매핑
3. SPH 4개 파일 → `sph_*` 테이블
4. 아정당 11개 파일 → `ajd_*` 테이블
5. 조인 매핑 검증 쿼리 실행

## 8. 주의사항 및 제약

### 데이터 제약
1. **SPH 파일명 오류**: 법정동마스터↔자산소득, 카드매출↔코드마스터가 뒤바뀜 → 적재 시 실제 내용 기준으로 매핑
2. **지리적 해상도 불일치**: SPH/리치고는 읍면동, 아정당은 시군구/시도 → 아정당 데이터는 상위 집계 후 조인
3. **리치고 범위 한정**: 서울 3개 구(중구, 영등포구, 서초구)만 → PoC는 이 3개 구로 한정하되, 확장 가능 구조로 설계
4. **시간 범위 차이**: SPH(2021~2025), 아정당(2023~2026), 리치고 시세(2012~2024) → 공통 기간(2023~2024) 중심 분석

### 스코어링 제약
1. 미개척도는 아정당 데이터가 시군구 단위이므로, 같은 구 내 법정동은 세대수 가중치로 차별화
2. 이사수요지수는 전세 시세 변동률의 프록시이며, 실제 이사 건수와의 상관관계는 별도 검증 필요
3. 1인가구 추정은 20~30대 비율 + 소형평형 거주 비율로 간접 추정 (직접 데이터 없음)
